# Taxonomy

> **НАЛАШТУВАТИ ПРИ ІНІЦІАЛІЗАЦІЇ ПРОЄКТУ.** Цей файл — дерево тем, за якими будуються `syntheses/`. Кожен лист → один файл синтезу. При появі нового листа — ДО створення синтезу — лист додається сюди з обґрунтуванням, чому це окрема тема, а не підрозділ наявної.

## Приклад структури

```
positioning/
├── messaging/          — меседжинг, ціннісні пропозиції
├── differentiation/    — чим відрізняємось від конкурентів
└── audience/           — сегменти, JTBD, болі

acquisition/
├── paid/               — платний трафік, CAC, ROAS
└── organic/            — контент, SEO, соцмережі

conversion/
├── landing-pages/      — структура й конверсія лендінгів
└── funnels/            — воронки, трипваєри, апсели
```

## Правила розширення таксономії

- Новий лист = нова тема, яку не можна розкрити в межах наявних синтезів без втрати ясності.
- Додавання листа фіксується комітом `taxonomy: add <path>` і коротким обґрунтуванням.
- Видалення/мердж листа припустимий, якщо після 20+ джерел тема не набрала 5 claim'ів — імовірно, це був не самостійний вузол.
