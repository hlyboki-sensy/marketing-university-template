# University — База знань проєкту {{PROJECT_NAME}}

Мета: **{{PROJECT_MISSION}}**. Усі знання тут підкріплені першоджерелами й придатні для подальшого зіставлення з реальним станом маркетингу проєкту (стратегія, офер, лендінг, кампанії, контент).

Це не звалище посилань. Це робоча база знань за правилами з [METHODOLOGY.md](METHODOLOGY.md).

## Структура

| Папка                | Що всередині                                                                                                                                |
| -------------------- | ------------------------------------------------------------------------------------------------------------------------------------------- |
| `sources/`           | Першоджерела «як є» — HTML/PDF/транскрипти. Одне джерело = одна папка `sources/<src-id>/` з `meta.yaml`, `original.*`, `conspect.md`.        |
| `notes/`             | Атомарні нотатки (Zettelkasten): одна ідея = один файл. Звʼязані перехресними посиланнями.                                                  |
| `claims/`            | Перевірювані твердження. Кожне — з доказовою базою, межами застосовності й імплікацією для проєкту.                                          |
| `syntheses/`         | Зведені документи за темами таксономії. До ~3000 слів на тему. Перезаписуються при появі нової інформації.                                   |
| `contradictions/`    | Зафіксовані суперечності між джерелами/твердженнями, що чекають на розвʼязання.                                                              |
| `verdicts/`          | Підсумкові рішення — що працює, що ні, чому.                                                                                                 |
| `experiments/`       | Власні тести (A/B, запуски, результати кампаній) для валідації claim'ів.                                                                     |
| `graph/edges.jsonl`  | Машиночитаний граф звʼязків (source → note → claim → synthesis).                                                                             |
| `queue/inbox.md`     | Куди користувач кидає нові посилання на обробку.                                                                                             |
| `queue/to-follow.md` | Внутрішня BFS-черга посилань, знайдених усередині вже оброблених статей.                                                                     |
| `queue/escalation.md`| Питання до користувача, на які Claude не може відповісти сам.                                                                                |
| `_templates/`        | Шаблони для нових джерел, нотаток, тверджень, синтезів.                                                                                      |
| `TAXONOMY.md`        | Що в скоупі, що поза — і дерево тем для синтезів.                                                                                            |
| `INDEX.md`           | Майстер-індекс усіх джерел/нотаток/тверджень зі статусами.                                                                                   |

## Як користуватися (оператору)

1. Кинь посилання в [queue/inbox.md](queue/inbox.md) або просто напиши в чат.
2. Claude обробляє за пайплайном з METHODOLOGY.md §4: fetch → relevance → quality → conspect → atomize → claims → recurse.
3. Періодично питаєш «що в нас по темі X» — Claude читає `syntheses/` і `claims/`, звіряє зі стратегією/офером/лендінгом, видає розбіжності (за процедурою §14).

## Верифікований вхід оператора

Окремо від зовнішніх посилань: будь-які **верифіковані дані, які ти подаєш на вхід** (deep research, аналіз аудиторій, готове дослідження, результати власних кампаній) — це найвищий клас джерела (`operator-verified`, §1.2 METHODOLOGY). Вони не проходять скоринг якості — ти вже за них ручаєшся — і стають **фундаментом**, з яким порівнюються зовнішні джерела.

## Ставлення до наявних документів проєкту

Університет **не замінює** наявну документацію проєкту. Якщо в проєкті вже є `research/`, `strategy/` чи аналог — ці папки обробляються як seed-джерела (див. [INDEX.md](INDEX.md) → Pre-existing internal sources) і додають згори нове знання із зовнішніх посилань.

Якщо нове знання з Університету суперечить наявному — це оформлюється як `contradiction`, а не тиха перезапис.
