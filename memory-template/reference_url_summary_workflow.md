**Якщо користувач кидає БУДЬ-ЯКИЙ URL (стаття, дослідження, YouTube, доки, тред), дія за замовчуванням: обробити його в `{{PROJECT_PATH}}/university/`.** Не переказ у чаті. Не ad-hoc відповідь. Повний пайплайн.

Пропустити пайплайн ЛИШЕ якщо користувач явно сказав «просто перекажи» / «не в університет» / «швидкий переказ».

## Що читати ПЕРШИМ

1. `{{PROJECT_PATH}}/university/README.md` — точка входу
2. `{{PROJECT_PATH}}/university/METHODOLOGY.md` — правила, читати перед кожним новим URL
3. `{{PROJECT_PATH}}/university/TAXONOMY.md` — фільтр in/out скоупу (§2)
4. `{{PROJECT_PATH}}/university/INDEX.md` — що вже оброблено (перевірка на дублі)
5. `{{PROJECT_PATH}}/university/_templates/` — використовувати ці шаблони дослівно

## Пайплайн (з METHODOLOGY.md §4) — кожен крок, по черзі

```
URL
 ├─ 1. Fetch        → sources/<src-id>/original.<ext> + meta.yaml
 ├─ 2. Relevance    → скоринг 0-5 проти TAXONOMY §2. Якщо 0 → sources/_rejected/
 ├─ 3. Quality      → скоринг 0-10 за чек-листом METHODOLOGY §3 (+ bias-прапорці)
 ├─ 4. Conspect     → sources/<src-id>/conspect.md (формат §6)
 ├─ 5. Atomize      → notes/n-<slug>.md × N (правила §7)
 ├─ 6. Claims       → claims/c-<NNNN>.md × M (формат §8, з conditions/durability/review_by)
 ├─ 7. Link extract → queue/to-follow.md (усі вихідні URL)
 ├─ 8. Synthesis    → оновити syntheses/<topic>.md, якщо картина змінилася (§9)
 ├─ 9. Graph        → дописати ребра в graph/edges.jsonl
 └─10. Index        → додати/оновити рядок в INDEX.md
```

## Верифікований вхід оператора (§1.2)

Якщо користувач подає не URL, а **верифіковані дані** (deep research, аналіз аудиторій, готове дослідження, результати власних кампаній) — це `source_class: operator-verified`: relevance 5, **не скориться** (§3 пропускаємо), заходить одразу на крок 4 (conspect), claim'и базово `confidence: H`. Це фундамент, з яким порівнюються зовнішні джерела.

## Конвенції ID (METHODOLOGY §11)

- Source: `src-YYYYMMDD-<slug>` (сьогоднішня дата, НЕ дата публікації)
- Note: `n-<slug>` (3-5 слів kebab-case, заголовок = твердження, не тема)
- Claim: `c-<NNNN>` zero-padded, наступний вільний з INDEX
- Synthesis: `syn-<taxonomy-leaf>`
- Contradiction: `contr-<NNN>`
- Verdict: `v-<slug>`

## Ідемпотентність

Перевір `INDEX.md` і canonical URL (прибрати utm_*, fbclid, ref= тощо) наявних `sources/*/meta.yaml` перед створенням нового джерела. Збіг → перевикористай наявний source_id.

## BFS глибина й ліміти (§5)

- **Дефолт — глибина 1-2** від кореневого URL користувача. Глибше (3+) — лише за явним «копай глибоко».
- **Дефолтна стеля — ~10-15 нових джерел** на корінь за сесію. Для повного deep research піднімаємо явно (до ~50).
- На кожній глибині йдемо лише за посиланнями з передбачуваною релевантністю ≥ 2.
- Дедуплікація через canonical URL + `graph/edges.jsonl`.
- **Свіжість**: тактики (платформи, формати, алгоритми) протухають за місяці; фундаментальні принципи — вічнозелені.

## YouTube окремо

Для YouTube-URL пайплайн:
1. Дістати транскрипт будь-яким доступним методом — по черзі:
   - Локальний скрипт (напр. `yt-dlp --write-auto-subs` або обгортка)
   - Пакет `youtube-transcript-api` (швидко, без авторизації, може лімітуватись)
   - Whisper на завантаженому аудіо (повільно, але надійно)
   - Якщо нічого не працює: попросити користувача вставити транскрипт вручну
2. Скопіювати транскрипт у `sources/src-YYYYMMDD-<slug>/transcript.txt`
3. Заповнити `meta.yaml` з `fetch_method: yt-transcript`, type: `youtube`
4. Побудувати `conspect.md` з транскрипту (цитувати тайминги, де критично)
5. Продовжити звичайний пайплайн з кроку 5 (atomize і далі)

## Стиль, який очікує користувач

Не нейтральний переказ. Структура як аналітичний бриф:

1. **Що джерело реально каже** — головна теза, ключові факти.
2. **Що по ділу** — claim'и, що витримують перевірку (з цифрами).
3. **Що маркетинг / хайп / непідтверджене** — назвати прямо. Підсвітити bias-прапорці.
4. **Твій критичний погляд** — чого бракує, що спрощено, для яких сегментів НЕ працює.
5. **Практичний висновок для користувача** — з огляду на те, що він будує.

Мова: **українська**. Тон: прямий, короткі речення, без води.
